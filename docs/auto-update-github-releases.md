# Auto-update via GitHub Releases (Tauri v2)

## 1. Generate signing keys

```bash
npm run tauri signer generate -- -w ~/.tauri/tradingbook.key
```

Keep private key safe. Never commit it.

## 2. Configure public key in Tauri

Replace `REPLACE_WITH_TAURI_UPDATER_PUBLIC_KEY` in `src-tauri/tauri.conf.json` with full public key content generated by Tauri signer.

Updater endpoint is configured to:

- `https://github.com/BMilc/TradingBook/releases/latest/download/latest.json`

## 3. Add repository secrets

In GitHub repository settings, add:

- `TAURI_SIGNING_PRIVATE_KEY`: private key content
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: private key password (empty if none)

## 4. Publish new version

1. Bump versions (`package.json`, `src-tauri/tauri.conf.json`, `src-tauri/Cargo.toml`).
2. Create and push tag `vX.Y.Z`.
3. Workflow `.github/workflows/release-tauri.yml` builds installers, signs updater artifacts, and uploads release assets.

## 5. Runtime behavior

At app startup, TradingBook checks updater endpoint in background.
If update exists, app downloads and installs automatically, then asks user to restart app.
